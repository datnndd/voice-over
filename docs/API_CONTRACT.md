# Voice Over API Contract

Contract version: `v1`

This contract is the stable surface for the frontend. Backend internals and `app_core` fields can evolve, but these request and response shapes stay compatible during Phase 1.

## Base Rules

- Content type: `application/json` for all request bodies.
- IDs: `job_id` is UUID string generated by backend.
- Timestamps: SQLite UTC strings in `YYYY-MM-DD HH:MM:SS` format.
- Secrets: never sent through API payloads; provider secrets come from `.env` / environment.
- Paths: response paths are absolute filesystem paths on the backend machine.
- Processing outputs default to `C:\Users\ddat2\Downloads` unless `VOICE_OVER_OUTPUTS` overrides it.
- Unknown provider/runtime fields must be treated as read-only metadata by frontend.

## Job Types

`type` must be one of:

- `stt`
- `tts`
- `subtitle_translate`
- `video_translate`

`tts` is the SRT-to-audio workflow. It accepts a local `.srt` or `.txt` path plus TTS provider settings and returns an audio file.

## Job Status

`status` must be one of:

- `queued`
- `running`
- `succeeded`
- `failed`
- `canceled`

## `POST /jobs`

Creates a queued job. Long processing is asynchronous.

### Request

```json
{
  "type": "video_translate",
  "params": {
    "name": "C:/Users/ddat2/Downloads/voice-over/test.mp4",
    "source_language": "en",
    "source_language_code": "en",
    "target_language": "vi",
    "target_language_code": "vi",
    "detect_language": "en",
    "subtitle_language": "vi",
    "recogn_type": 10,
    "model_name": "nova-3",
    "translate_type": 3,
    "tts_type": 28,
    "voice_role": "HoaiMy(Female)",
    "app_mode": "biaozhun",
    "subtitle_type": 0,
    "output_srt": 0,
    "recogn2pass": false,
    "fix_punc": false,
    "stt_punctuate": true
  }
}
```

### Stable `params` Fields

| Field | Type | Direction | Notes |
| --- | --- | --- | --- |
| `name` | string | request | Source media/subtitle path. |
| `source_language` | string | request | Display or language name/code. |
| `source_language_code` | string | request | Source language code, e.g. `en`. |
| `target_language` | string | request | Display or language name/code. |
| `target_language_code` | string | request | Target language code, e.g. `vi`. |
| `detect_language` | string | request | STT language hint. |
| `subtitle_language` | string | request | Subtitle embed language. |
| `recogn_type` | integer | request | STT provider id from `/providers`. |
| `model_name` | string | request | STT model name. |
| `translate_type` | integer | request | Translator id from `/providers`. |
| `tts_type` | integer | request | TTS provider id from `/providers`. |
| `voice_role` | string | request | TTS voice display name. |
| `app_mode` | string | request | Runtime mode, default `biaozhun`. |
| `subtitle_type` | integer | request | Subtitle embed mode. |
| `output_srt` | integer | request | Subtitle output mode. |
| `recogn2pass` | boolean | request | Re-recognize dubbed audio. |
| `fix_punc` | boolean | request | Restore punctuation after STT for zh/en. |
| `stt_punctuate` | boolean | request | Enable provider-native punctuation when STT supports it. |
| `enable_diariz` | boolean | request | Enable speaker diarization. |
| `nums_diariz` | integer | request | Speaker count hint, `0` means auto/no limit. |
| `speaker_clone_mode` | string | request | `off` for single voice, `auto` for diarize + auto OmniVoice refs. |
| `speaker_ref_min_seconds` | integer | request | Minimum reference audio seconds per speaker, default `10`. |
| `speaker_ref_max_seconds` | integer | request | Maximum reference audio seconds per speaker, default `15`. |
| `uuid` | string | response | Backend job id injected into runtime. |
| `target_dir` | string | response | Job output directory. |
| `dirname` | string | response | Source parent directory. |
| `basename` | string | response | Source filename. |
| `noextname` | string | response | Source filename stem. |
| `ext` | string | response | Source extension without dot. |
| `targetdir_mp4` | string | response | Copied source video path, named `video_renew.<ext>`, for video jobs. |

`video_translate` outputs are independent assets. The backend does not mux translated voice into the returned video. Expected successful outputs are:

- `source.srt`: source-language transcript.
- `target.srt`: translated subtitle.
- `voice_target.m4a`: target-language voice output.
- `video_renew.<ext>`: direct copy of the source video.

Extra fields are accepted as runtime pass-through for `app_core`, but frontend should rely only on stable fields above.

### SRT-to-audio `tts` request

```json
{
  "type": "tts",
  "params": {
    "name": "C:/Users/ddat2/Downloads/script.srt",
    "target_language": "vi",
    "target_language_code": "vi",
    "tts_type": 28,
    "voice_role": "HoaiMy(Female)",
    "voice_autorate": false,
    "remove_silent_mid": false,
    "align_sub_audio": false
  }
}
```

Expected successful output:

- `<srt-stem>.wav`: synthesized audio from subtitle text and timing.

Frontend rule:

- Provide a separate `SRT → audio` mode in the create-job form.
- Require only SRT path, audio language, TTS provider, and voice role.
- Submit `type: "tts"`; do not require STT or translator provider readiness for this mode.

### Response `202`

```json
{
  "id": "fffb497b-c8c1-4195-b5d2-a187bc9a9960",
  "type": "video_translate",
  "status": "queued",
  "params": {},
  "error": null,
  "target_dir": "C:/Users/ddat2/Downloads/fffb497b-c8c1-4195-b5d2-a187bc9a9960",
  "created_at": "2026-06-07 10:43:52",
  "updated_at": "2026-06-07 10:43:52"
}
```

    {

      "filename": "video_renew.mp4",
      "extension": ".mp4",
      "kind": "video",
      "size_bytes": 123456
    },
    {
      "path": "C:/Users/ddat2/Downloads/.../voice_target.m4a",
      "filename": "voice_target.m4a",
      "extension": ".m4a",
      "kind": "audio",
      "size_bytes": 123456
    },
    {
      "path": "C:/Users/ddat2/Downloads/.../source.srt",
      "filename": "source.srt",
      "extension": ".srt",
      "kind": "subtitle",
      "size_bytes": 1234
    },

      "extension": ".srt",
      "kind": "subtitle",

`kind` must be one of:

- `subtitle`: `.srt`, `.vtt`, `.ass`
- `audio`: `.wav`, `.mp3`, `.m4a`, `.aac`, `.flac`, `.ogg`
- `video`: `.mp4`, `.mov`, `.mkv`, `.webm`
- `other`: everything else

### Response `200`

{
  "stt": [
    {
      "id": 10,
      "name": "Deepgram.com",
      "kind": "stt",
      "runtime": "api",
      "key_name": "deepgram_apikey",
      "env_var": "VOICE_OVER_DEEPGRAM_APIKEY",
      "configured": true,
      "config_mode": "env"
    }
  ],
  "tts": [
    {
      "id": 32,
      "name": "VieNeu-TTS",
      "kind": "tts",
      "runtime": "local",
      "key_name": null,
      "env_var": null,
      "configured": true,

## Multi-Speaker Auto Clone

Frontend mode `Nhi?u ng??i n?i ? auto clone` submits:

```json
{
  "enable_diariz": true,
  "nums_diariz": 0,
  "tts_type": 2,
  "voice_role": "clone",
  "speaker_clone_mode": "auto",
  "speaker_ref_min_seconds": 10,
  "speaker_ref_max_seconds": 15
}
```

Backend behavior:

- Run STT and diarization to produce `speaker.json`.
- Group source subtitle lines by `spkX`.
- Cut and concatenate 10-15s reference audio per speaker under `speaker_refs/`.
- Write `speaker_refs/speakers.json`.
- Inject `ref_wav` and `ref_text` into each OmniVoice TTS queue item based on speaker label.
- If diarization refs are unavailable, fallback to existing per-line clone reference behavior.

## `POST /voices/clone-refs`

Uploads a reference audio sample for clone-capable local TTS providers such as OmniVoice and VieNeu-TTS. The backend stores the file under `f5-tts/`, registers it in runtime voice config, and returns the new voice role name.

### Request

Multipart form data:

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `file` | file | yes | Reference audio, e.g. `.wav`, `.mp3`, `.m4a`. |
| `ref_text` | string | yes | Exact text spoken in the reference audio. |

### Response `200`

```json
{
  "name": "sample.wav",
  "path": "C:/.../f5-tts/sample.wav",
  "ref_text": "hello sample"
}
```

Frontend rule:

- Show this upload UI as a separate voice library for clone-capable TTS providers such as `tts_type=2` OmniVoice and `tts_type=32` VieNeu-TTS.
- After upload, refresh `/voices?tts_type=2&language=...` or `/voices?tts_type=32&language=...` and select returned `name` as `voice_role` for single voice.
- Do not use `voice_role=clone` unless the core pipeline supplies per-line source references automatically.

## `GET /voices`

Returns selectable TTS voices from the runtime voice JSON/config used by the original project.

### Query

| Field | Type | Required | Notes |
| --- | --- | --- | --- |
| `tts_type` | integer | yes | TTS provider id from `/providers`. |
| `language` | string | no | Target language code, e.g. `vi`, `en`, `ja`. |

### Response `200`

```json
{
  "tts_type": 28,
  "language": "vi",
  "voices": [
    {
      "name": "HoaiMy(Female)",
      "value": "HoaiMy(Female)",
      "language": "vi",
      "gender": "female"
    }
  ]
}
```

Frontend rule:

- Azure-TTS returns language-specific voices from `azure_voice_list.json`.
- OmniVoice returns clone/reference voice roles from local runtime config.
- `name` is display text; `value` is sent back as `voice_role` in job params.

## `GET /runtime/checks`

Reports runnable status for providers and system dependencies.

### Response `200`

```json
[
  {
    "provider_kind": "stt",
    "provider_id": 10,
    "provider_name": "Deepgram.com",
    "status": "ready",
    "missing": []
  },
  {
    "provider_kind": "system",
    "provider_id": null,
    "provider_name": "ffmpeg",
    "status": "ready",
    "missing": []
  }
]
```

Frontend rule:

- `status="ready"` and `missing=[]`: selectable.
- `status="missing"`: show disabled provider and list `missing` values.
